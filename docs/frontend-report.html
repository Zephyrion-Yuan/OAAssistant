③ deep-chat 选型 & 集成

按「先找可直接用的开源框架」原则选了 deep-chat(MIT,cdn.jsdelivr.net/npm/deep-chat@2,框架无关 Web Component,CDN 免构建)。它给了 polished 的对话外壳(头像/气泡/输入框/自动滚动/流式),正好去掉「程序式」感。

// 自定义元素 upgrade 后再配置;handler 跑我们自己的 BFF SSE,
// 用 onResponse({html, overwrite:true}) 把进度 chips + 草稿卡渲染进同一个生长气泡
customElements.whenDefined('deep-chat').then(() => {
  dc.connect = { stream: true, handler: (body, signals) => chatHandler(body, signals) };
  dc.history = [{ role: 'ai', html: intro }];
});
// chatHandler: fetch BFF /api/chat 流 → 逐 data: 事件 → signals.onResponse({html, overwrite})
一个关键约束:deep-chat 的 html 消息渲染在它的 shadow DOM 里,页面全局 CSS 进不去,所以草稿卡用内联样式(每个卡片自带 style)而非 class。右半填表 + 配置抽屉仍由 Vue 管理。

⑤ 怎么跑(三个独立进程)

# 1) Node「手」(WBS registry / 真机填单;离线也建议起,供真实 WBS)
MEGANT_EDGE_PROFILE_MODE=sso-handoff npm start                      # :8787

# 2) BFF 网关(需 orchestrator/.env 的 DEEPSEEK_API_KEY —— LLM 必需)
PYTHONPATH=orchestrator orchestrator/.venv/bin/uvicorn oa_orchestrator.bff:app --port 8788

# 3) 前端(任意静态服务器,真·解耦)
cd frontend && python3 -m http.server 5500       # 打开 http://127.0.0.1:5500
LLM 必需:聊天的 classify_goal 一定调 DeepSeek(无 fallback),mock 模式只是把 OA/PDM/库存换成假后端,LLM 仍是真的。
