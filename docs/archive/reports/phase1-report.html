② 分配 + 分桶算法(纯函数,确定性,无 LLM)

对每个需求行(物料 + WBS + 数量 Q),按优先级 412 > 89 > 458 分配;一个物料的库存池跨行扣减(两行同物料不会重复占用库存)。

# 每物料库存池:public(公共仓,SOBKZ空) + project{WBS: 量}(专属仓,SOBKZ=Q)
own  = pool.project[demandWBS]                  # 本项目专属仓 → 可直接出库
a412 = min(Q, public + own)                     # 412 出库(扣 own 再扣 public)
rem  = Q - a412
for srcWBS, avail in 其它项目专属仓(按量降序):       # 89 转储,每个源 WBS 一张草稿
    a89 = min(rem, avail); rem -= a89           #   转出=srcWBS, 转入=demandWBS
a458 = rem                                      # 458 采购缺口

分桶键:412 = (412, WBS);89 = (89, WBS, 源WBS);458 = (458, WBS)。跨物料同键合并物料行 → 每个桶 = 一张草稿。

库存情形路由移动类型(89)
公共仓 / 本项目专属仓412 出库—
别项目专属仓(SOBKZ=Q)89 转储项目库存转储至项目库存(自动推导)
不足 / 无库存的缺口458 采购—

④ 关键文件代码解读

4.1 nodes/route_workflow.py — 分配 + WBS 分桶(核心)

# 跨需求行扣减同一物料的库存池;每个正分配进对应 (流,WBS) 桶
own = pool["project"].get(wbs, Decimal("0"))
a412 = min(need, own + pool["public"])
pool["project"][wbs] = own - min(a412, own)        # 先扣 own
pool["public"] -= (a412 - min(a412, own))          # 再扣 public
if a412 > 0: _add_line(bucket(("412", wbs), "412", wbs), material, name, a412, unit, meta)
remaining = need - a412
for src_wbs, avail in others:                        # 89:每个源 WBS 一张草稿
    a89 = min(remaining, avail); remaining -= a89
    if a89 > 0: _add_line(bucket(("89", wbs, src_wbs), "89", wbs, src_wbs), ...)
if remaining > 0:                                   # 458 缺口
    _add_line(bucket(("458", wbs), "458", wbs), material, name, remaining, ...)

⑦ 已知边界 & 下一步

需对真实数据确认

  • 458 附件列模板:目前用一套假定列(需求日期/项目定义/WBS/工厂/物料/数量/单位/MRP)。上真机前要拿一份真实 458 附件样例核对列名/顺序。
